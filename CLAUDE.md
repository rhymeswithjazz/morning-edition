# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Daily curated tech magazine at **daily.rhymeswithjazz.com**. Each edition has 20 stories (10 Hacker News, 10 Pinboard Popular) rendered as a single-page HTML magazine with 20 distinct visual themes. Deployed via GitHub Pages on push to main.

## Build Commands

```bash
# Full daily pipeline (what .github/workflows/daily-edition.yml runs)
python3 scripts/fetch-stories.py                                       # → magazines/candidates-YYYY-MM-DD.json
python3 scripts/curate-edition.py magazines/candidates-YYYY-MM-DD.json # → magazines/YYYY-MM-DD.json (requires ANTHROPIC_API_KEY)
python3 scripts/publish-edition.py magazines/YYYY-MM-DD.json           # build + commit + push

# Individual steps (used by publish-edition.py internally)
python3 scripts/build-edition.py magazines/YYYY-MM-DD.json    # render HTML from JSON
python3 scripts/build-index.py                                # rebuild homepage, archive, RSS, redirect

# Utilities
python3 scripts/recent-urls.py [days]                         # dedup helper (default 7 days)
python3 scripts/publish-edition.py FILE --dry-run             # build only, no git
python3 scripts/publish-edition.py FILE --no-push             # commit but don't push
```

No dependencies beyond Python 3 standard library.

## Data Flow

```
scripts/fetch-stories.py                    (fetch HN + Pinboard, dedup)
        ↓
magazines/candidates-YYYY-MM-DD.json        (raw candidates for curation)
        ↓  scripts/curate-edition.py  (Claude API, prompts/curation-system.md)
magazines/YYYY-MM-DD.json                   (curated 20 stories)
        ↓  scripts/publish-edition.py
magazines/YYYY-MM-DD.html + index.html + archive + feed.xml + git push
```

The GitHub Actions workflow at `.github/workflows/daily-edition.yml` runs this pipeline on cron at 08:00 UTC (4 AM EDT / 3 AM EST) and on `workflow_dispatch`.

## Architecture

- **scripts/fetch-stories.py** — Pre-fetcher. Pulls HN top 40 (parallel via ThreadPoolExecutor) and Pinboard Popular RSS, deduplicates against last 7 days, writes candidates JSON.
- **scripts/curate-edition.py** — Curator. Calls the Claude API (Sonnet 4.6 by default, override via `CURATOR_MODEL`) with `prompts/curation-system.md` as the system prompt and the candidates JSON as the user message. Uses a forced `submit_edition` tool call so the response shape is guaranteed. Writes `magazines/YYYY-MM-DD.json`. Requires `ANTHROPIC_API_KEY`.
- **scripts/publish-edition.py** — Publisher. Validates curated JSON, runs build-edition + build-index, commits and pushes to main. Supports `--dry-run` and `--no-push`.
- **scripts/build-edition.py** — Template engine. Takes a JSON file, produces a full HTML page with inline CSS. Each of the 20 stories gets a unique visual "spread" theme (10 HN styles, 10 Pinboard styles). All styles are embedded inline — no external CSS.
- **scripts/build-index.py** — Site generator. Scans `magazines/*.html`, extracts metadata from rendered HTML, produces homepage, archive, RSS feed, and redirect page.
- **scripts/recent-urls.py** — Standalone dedup helper. Reads JSON files from recent days, outputs URLs. Logic is also inlined in fetch-stories.py.
- **prompts/curation-system.md** — Editorial instructions, reader profile, interest categories, and blurb guidelines. Loaded as the system prompt by `curate-edition.py`.

## JSON Schema for Editions

```json
{
  "date": "YYYY-MM-DD",
  "stories": [
    {
      "title": "Headline",
      "url": "https://...",
      "hn_link": "https://news.ycombinator.com/item?id=...",
      "category": "AI Tools",
      "applies": true,
      "byline": "by author · 500 points",
      "blurb": "2-3 sentence editorial summary"
    }
  ]
}
```

- Stories 1–10: Hacker News (include `hn_link`)
- Stories 11–20: Pinboard Popular (omit `hn_link`)
- `applies`: flags stories relevant to the reader (dev tools, Apple ecosystem, privacy, Claude Code)

## Deployment

The scheduled GitHub Actions workflow (`.github/workflows/daily-edition.yml`) runs the full fetch → curate → publish pipeline on cron at 08:00 UTC and on manual dispatch. It commits directly to main; GitHub Pages auto-deploys. On any failure it POSTs a Slack webhook with a link to the run log.

Required repo secrets: `ANTHROPIC_API_KEY`, `SLACK_WEBHOOK_URL`.

Manual runs (local) follow the same three commands in Build Commands. Push directly to main — no branches, no PRs. Commit message format: `Morning Edition — {DayOfWeek}, {Month} {Day}, {Year}`.

## Key Conventions

- All HTML/CSS is generated and inline — never hand-edit rendered HTML files
- The homepage (`index.html`) is always a copy of the latest edition, generated by `build-index.py`
- Fonts: Fraunces (serif) + Inter (sans-serif) loaded from Google Fonts
- No npm, no bundler, no build tools beyond Python
